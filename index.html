Arduino 1 -- remote-controlled car

After I learnt some basic knowledge about Arduino, I did a remote-controlled car with the help of some videos on the Internet and this is a record of how I made this car. The whole record will be broken into two parts. The first part aims to show how I put different pieces together and the second part will the codes used in this car.

This is the part of putting pieces together. First I will show the final product.

The whole car can be broken into three parts. Next, I will introduce them one by one.

——Mainboard

Mainboard is the most difficult part to weld because there are so many small pieces in it. Before we weld it, it is empty.

After I weld it, it is filled with small components.

We can know what to weld by looking at the tips on the empty board. There are many patterns on the empty board and just find the component that follows the pattern. Do not forget that some components have positive and negative sides. And fix component well before you weld it or it will look ugly.

After welding small pieces , we should put the mainboard and UNO board together and fix them to the bottom board of the car.

——Sensor part

Sensor part aims to achieve some functions of this car such as making the car move following a path and making the car avoid barriers automatically. First, following path is achieve by a infra-red

sensor. It can emit infra-red light and receive the reflected light to determine the color of the road.

The next part is avoiding barriers. The principle of this part is similar to that of following path. It also used infers-red light to calculate the distance between barrier and it self to avoid barrier.

——battery and wheel part

This is the most easy part to do because there is no special tips in this part. However, it is the most important part because it is the power of the whole car.

That is all for the three parts and finally we just need to connect wires between different parts.

​ Overall, this book is good. I think this book is deeper in calculus than the joy of X. The

​ My capstone of X Academy

​ In X Academy, everyone should do a project using what they learned here. In this article, I will discuss what I did in my capstone.

​ My program in X Academy was called Data Science so my capstone was related to processing data. I found my data set in a huge data set website called Kaggle. This data set was about the potability of water. It gave me a few properties of a kind of water such as ph, hardness, solids and I needed to use these properties to find whether this kind of water was potable.

​ First, I would show the first five rows of my data set.

​ By looking at the data, we could find that some data was lost in this data set so we needed to fill these lost data. The picture showed how many lost data are there in this data set.

​ For this purpose, I chose to use the mean of the whole column to fill it as the mean of the whole column could make fewer errors. I took ph as example to show my codes.

​ After dealing with lost data of ph, sulfate and trihalomethanes, all lost data was filled again.

​ Besides lost data, we could also find that some data in this data set was very unreasonable. For example, the ph of a kind of water was 2 but the data set still said that this water was potable. To get an accurate result, we needed to drop these data. Take ph as example, we should know the right save range for ph first, which was 6 to 9. Then, we checked the ph values of all kinds of water one by one. If the ph of this kind of water was lower than 6 or higher than 9 and the data set still said it was potable, we would drop the whole row of this water as this row of data was unreasonable.

​ After doing this, the whole data was cleaned completely. The next step was to choose a model to analyze my data. I would like to tell an interesting story in this step. At first, I did not realize that this was a classification question so I chose a lot of regression models which was a waste of time. However, maybe regression was not completely useless. If I rounded the value predicted by the regression model, I may get a result which was similar to that got from classification models. In fact, I tried many classification models but there was a problem. Many models predicted that all water was not potable in this data set. I checked my codes for many times and I was sure that my codes were correct so I thought this was the problem of the data set which means the values in this data set were not very correct. Finally, I decided to use decision tree as my classification as it can both output 1 and 2. The picture below was my code.

​ So far, I had finished most of my code. The last step was to check the results predicted by my model. First, we can see that the accuracy of this model was only 77 precent which means the result of this model was not so good.

​ The picture above showed the specific result predicted by the model. We can find that, for potable water, the accuracy of my model was 50 precent. However, for unpotable water, the accuracy was 85 percent. So for this code, it had better be used to predict whether the water was unpotable instead of using it to predict whether the water was potable. In another words, if the model says the water is unpotable, you can trust it. However, if the model says the water is potable, you had better do not drink it.

Banqioa project

​ The report of Banqiao project (meeting 1)

​ On July 2, we went to Banqiao factory again. This time, we were able to take a look of a sample machine which was used to remove burrs on some particular components. Before this step was done by workers. However, the working environment of this job was very hard as this job was boring and workers would work in smoke. As a result, it was hard to employ workers and the cost of employing workers was high. To solve this problem, people wanted to use machine to replace most workers and just employed a few people to control it. In this article, I wanted to introduce the sample machine. Actually, the machine that I talked about this time was not the final product. It was just a sample to show how this kind of machine worked.

​ First, I wanted to show a general appearance of the sample machine.(figure1.1 1.2)

figure 1.1

figure 1.2

​ Then, I would talk about it part by part. The first part was a holder to place components before machine worked. (figure 2.1)

figure 2.1

​ In this holder, there were several supports to hold cylinder components. If the component had a different shape, we just needed to change the shape of supports and change the code of the machine. Under each support, there was a weight sensor which was used to check whether the component was placed well. Workers were needed to place components to the holder so workers could not be replaced completely. But this did not matter, the job was much better than the job which was needed to remove burrs.

​ The machine arm was also very important as cutters were fixed in this machine. It was the machine arm that controlled the components and put them towards the cutters. Moreover, burrs almost appeared on the same places of each component because burrs was caused by a few particular reasons which were same for all components. As a result, the machine arm just needed to repeat several steps. Also, you could find that the front part of the machine arm was much thinner as the part moved many times so it should be flexible. The latter part of the machine arm was much thicker as this part should be stable so that the whole machine would not shake easily. (figure 3.1)

figure 3.1

​ The last part was very simple – cutters and sand paper. Cutters were used to remove bigger burrs and sand paper was used to remove smaller burrs to make the component flatter.
