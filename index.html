Object-oriented or Process-oriented

Object-oriented or process-oriented

This is common question asked by program learner. Here, I will give my personal idea about it. Object-oriented means it emphasis on each individual. It looks what you can do. Process-oriented emphasis on the steps of task. For example, there is an exam. If you concern what does teacher do and what does candidate do, this is object-oriented. However, if you concern, what should be done first and what is next, this is process-oriented. Commonly, people are used to process-oriented thinking.

Then, I want to discuss about the advantages and disadvantages of these two concept. For process-oriented, I think the main advantage is that it is easier to be understood as it is similar to human’s thinking. But the disadvantage is that all things are linked together and it is hard to replace any element in it. however, for object-oriented, it is completely opposite. It is easily to change but hard to understand. If you check an object-oriented program, you will find that different functions and procedures are everywhere and it is hard to understand. The only method to reduce this is writing comments. As a result, if you just write a small amount of code, you can write in process-oriented as it is fast and is easily understood. However, if you write a huge program, you had best to write in object-oriented as there are many maintenances later. Using object-oriented is easier to be modified. But you can also write process-oriented to smooth your thoughts then go back to object-oriented.

Why I want to talk about this topic. Because, when I learnt how to use Python to write code, the teacher firstly wrote process-oriented code to make us understand and then change it into object-oriented code to fulfill it. at first, I did not understand why the teacher taught in this way util I did some research about it and this topic really activates my thoughts about how to write code in a more efficient way.
